Brand Guide

Introduction

AllyMap is our MultiMediaProject 2A and this is its Styleguide.

The main goal is a Web-App, which provides a map, where you can see the different accessibillity levels of public places regarding wheelchair usage. The secondary goal is to give users the possibility to plan routes with these points of interests.

CSS Methodologies

Code Style Rules

  • Use 4 Spaces for Indentation
  • Use Semicolons
  • Each reusable component has its own styling file in the styles folder
  • All components have to be imported into one main.scss
  • Use Normalize.css as a css reset
  • Use SCSS as a preprocessor
  • Use PostCSS Autoprefixer as a postprocessor
  • Using rem and % for sizing is encouraged
  • Use BEM Syntax
  • Use SCSS variables or CSS Custom Properties for things you need more often
  • Use camelCase for variables

Naming Conventions

We use BEM. BEM stands for Block Element Modifier and is a methodology that helps you to create reusable components and code sharing in front-end development. BEM and SCSS work hand in hand because the styles can be very well structured like in the example below.

A block is a independant element, which can be used anywhere it is needed, without to change anything. The name of the block should describe what it is e.g. 'navbar' or 'infobox'. If it consits of more than one word, they are connected with a dash e.g. 'main-navigation'

An element is a element, which is dependant on the block, which it is nested into. The name consists of the block name, which it belongs to and its own name. Those two halfes are connected by two underscores e.g. list__item or color__value.

A modifier is a class, which changes a detail of the element, which it is applied to. Its name is built by appanding the modification to the block or element name with two dashes e.g. color--green or font__item--large.

For further reading check out its homepage HERE.

Example


.block { }
.block__element { }
.block--modifier { }
.block__element--modifier { }

Real Example with use of SCSS


.main-navigation {
    &__list {
        list-style-type: none;

        }
    }
}
